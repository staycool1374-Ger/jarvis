Detailed Description

Reusable diagnostic harness for IPC reply-loss (H1) and deferred- switch self-deadlock (H2) analysis in the scheduler/IPC subsystem.

Everything here is gated behind CONFIG_DEBUG_IPC_SCHED. Flip the #define in kernel/jarvis_config.h (or uncomment below) to re-enable the whole proof kit for future analysis iterations without re-deriving the instrumentation.

Proven signatures (what each trace tag proves): [SYNC] entered IPC::send_sync for (cur -> dest) [SEND]/[WAKE] a message was pushed into dest's queue and dest was woken [SYNC-FAIL] dest-gone-empty -> genuine peer death, reply never arrived (OK) [SYNC-FAIL] dest-gone-reply -> H1 PROVEN: reply present in own queue but send_sync returned false and discarded it [WEDGE] H2 candidate: save_rsp_to != 0 while a READY task is orphaned (inrq=1, phys=0) and the ready bitmap is empty -> the timer ISR's rate_monotonic_schedule early-returned on a stale pending-switch flag and never re-dispatched the driver, so the driver could never clear the flag -> self-deadlock.
