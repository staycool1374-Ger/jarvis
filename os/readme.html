Jarvis RTOS

A real-time operating system for the x86_64 architecture, written from scratch in C++. Designed with hard real-time constraints, deterministic scheduling (Rate Monotonic), and clean userspace isolation.

Features

Current Version: 0.2.6-dev

Core System & Memory Management

  • x86_64 Long Mode Boot – Fully Multiboot2 compatible bootloader support.
  • Physical Memory Manager (PMM) – Fast page frame allocation via a Buddy Allocator.
  • Virtual Memory Manager (VMM) – Full 4-level page table hierarchy managing kernel and user spaces.
  • Slab Allocator (MemPool) – Object cache layer for deterministic kernel allocations.
  • Heap Allocator – Dynamic kernel heap with OOM (Out Of Memory) handling and fallback mechanisms.

Process & Task Management

  • Rate Monotonic Scheduler (RMS) – Priority-based preemptive scheduling for deterministic real-time tasks.
  • Robust Task State Machine – Full support for READY, RUNNING, BLOCKED, WAITING, and TERMINATED states.
  • Userspace Isolation – Ring 3 execution with dedicated user stacks and page protection.
  • Process Lifecycle Control – Native fork() and waitpid() implementation for UNIX-like process trees.

Interprocess Communication (IPC)

  • Priority-aware MessageQueue with per-TCB mailbox and priority bitmap.
  • IPC primitives: SEND, RECEIVE, SEND_SYNC, IPC_NONBLOCK.
  • Priority inheritance for IPC operations prevents priority inversion.
  • Linked blocked-sender list for O(1) sender unblocking.
  • Task Notification (Notify) – Lightweight notification value signaling via SYS_NOTIFY/SYS_NOTIFY_WAIT.
  • Event Group – Multi-bit event synchronization via SYS_EVENT_SET/SYS_EVENT_WAIT.

Virtual File System (VFS)

  • Unified vnode abstraction layer.
  • Per-task file descriptor tables tracking standard I/O and custom files.
  • Implemented Filesystems:
    • Initrd – Initial ramdisk parsed from the standard POSIX cpio (newc) format.
    • Devfs – Device abstraction mapping /dev/tty, /dev/null, /dev/console, and /dev/kbd.
    • Procfs – Kernel metrics interface exposing /proc/meminfo, /proc/[pid]/stat, and /proc/self.

Userspace Ecosystem

All utilities are built against Jarvis' minimal custom libc:

  • sh – Interactive userspace shell with full pipe support (|), output redirection (>), input redirection (<), and builtins (cd, export).
  • hello – Standard testing binary compiled as a static ELF executable.
  • Automatic loading from initrd into Ring 3 environments.

Debugging & Diagnostic Tools

  • Built-in benchmark suite (CPU operational stress tests, memory allocator bottlenecks).
  • Integrated self-test framework executing over 416 structural test cases.
  • Kernel command: selftest [category] to isolate validation routines.

Build Requirements

Emulation Environment

  • QEMU (specifically qemu-system-x86_64)
  • Allocated testing memory: >= 512 MB RAM

Linux Toolchain Installation

sudo apt update && sudo apt install -y \
    build-essential git wget ninja-build xorriso dosfstools \
    grub-pc-bin iasl gcc-x86_64-linux-gnu binutils xz-utils qemu-system-x86

Compiler Dependencies

  • Cross-Compiler: x86_64-linux-gnu-gcc >= 7.0 (configured for static, freestanding linking)
  • Build System: GNU Make
  • Utilities: objcopy, ld from GNU Binutils

Setup & Installation

1. Clone the repository

git clone [repository-url]
cd jarvis/os

2. Compile the System

Build the kernel binary, compile the userspace applications, and pack the initrd into a bootable ISO image:

make all

3. Execution via QEMU

Launch the compiled ISO with isolated hardware profiling and mapped serial monitor:

qemu-system-x86_64 -m 512M \
    -drive format=raw,file=jarvis-rtos.iso \
    -serial mon:stdio

Alternatively, you can build and launch directly in one step:

make qemu-iso

Usage Examples

Kernel Boot Shell

Upon boot, the kernel lands in an initial setup prompt:

boot> ls        # List registered kernel structures, tasks, and drivers
boot> sh        # Spawn the interactive Ring 3 Userspace Shell

Userspace Shell Environment

Once inside the user terminal, standard UNIX abstractions apply:

~/ ls
~/ cd /dev
~/ export MY_VAR=value
~/ echo $MY_VAR
~/ cat /proc/meminfo
~/ hello | cat > /tmp/output.txt

The 6-Step Feature Iteration Cycle

  1. Implement: Write or expand system features.
  2. Test Design: Add comprehensive units to the local test suite.
  3. Validate Locally: Execute make tests to ensure code safety.
  4. Regression Check: Verify all 416 automated test metrics report green.
  5. Release Build: Compile the final image using make release.
  6. Document: Update the code inline (Doxygen) and track metrics in ROADMAP.md.
